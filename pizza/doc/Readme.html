INTRODUCTION

The Pizza Party Planner application is a simple 8 agent society that exercises many key Cougaar modules, while staying relatively simple. It demonstrates a little of what Cougaar can do, and key features. It is also a good test to ensure key pieces of Cougaar are still working, and a good set of sample code for modeling custom components off of.

INSTALLATION

  1. Verify that Java 6 (aka 1.6) is installed and is on the PATH:
      java -version

  2. Create a Cougaar install directory, e.g.:
      mkdir /usr/local/cougaar
    Set this directory as the COUGAAR_INSTALL_PATH environment variable:
      export COUGAAR_INSTALL_PATH=/usr/local/cougaar

    This directory can optionally be shared and read-only.

  3. Download Cougaar 12.6 or later.
    The latest release can be found at:     Downloads.

  4. Unzip the "cougaar.zip", "cougaar-support.zip", and (optional-- see below) "demo-pizza.jar" into the COUGAAR_INSTALL_PATH.

  5. Optionally add the "$COUGAAR_INSTALL_PATH/bin/cougaar" script to the command-line search PATH:
      export PATH=$COUGAAR_INSTALL_PATH/bin:$PATH

    This can also be done by adding the following line to your "~/.bashrc":
      function cougaar() { $COUGAAR_INSTALL_PATH/bin/cougaar $* ; }

  6. Create a per-user Cougaar "user" directory, e.g.:
      mkdir ~/cougaar

    Applications and demos will go into this directory.

    This directory can be the same as the COUGAAR_INSTALL_PATH, but (in general) it is cleaner to keep these directories separate.

    The Cougaar "user" directory will be the Eclipse "workspace" directory.

  7. Move the "pizza/" directory into your Cougaar "user" directory.

    If you're using the "demo-pizza.zip" from the release, then unzip this zip into your Cougaar "user" directory.

    If you're using Git, then checkout the master branch of the "pizza" module into your Cougaar "user" directory:
      git clone http://cougaar.org/git/tutorials.git

  8. Set the COUGAAR_SOCIETY_PATH environment variable:
      export COUGAAR_SOCIETY_PATH=~/cougaar/pizza

    The COUGAAR_SOCIETY_PATH will be the Eclipse "project" directory.

  9. Set the COUGAAR_RUNTIME_PATH environment variable:
      export COUGAAR_RUNTIME_PATH=$COUGAAR_SOCIETY_PATH/run

    This directory will contain the "runtime" XML file, which defines the Java System Properties, and will contain any output logs and other per-run temporary files.

    For simplicity, this directory is packaged as the "run/" subdirectory of the COUGAAR_SOCIETY_PATH. Users can move and rename this "run" directory to any location, so long as the COUGAAR_RUNTIME_PATH environment variable is set correctly and the directory is writable.

    The COUGAAR_RUNTIME_PATH will be the Eclipse "working" directory.

  10. Compile the source code to create the "lib/pizza.jar":
      ant dist

    If you've unzipped the "demo-pizza.zip" then this step is optional. The zip includes a pre-compiled "lib/pizza.jar".

SETUP

Ensure that you installed cougaar.zip and cougaar-support.zip and that your COUGAAR_INSTALL_PATH environment variable points to the directory where those were unzipped. It will also help if you put COUGAAR_INSTALL_PATH/bin on your PATH.

RUNNING

To run this application, you run the 2 Cougaar Nodes that make up the system:

[In a command line window]
% cd COUGAAR_INSTALL_PATH/pizza/configs/pizzaparty
% cougaar SDPizzaNode1.xml

[Start another window on the same machine]
% cd COUGAAR_INSTALL_PATH/pizza/configs/pizzaparty
% cougaar SDPizzaNode2.xml

OTHER OUTPUT

As with all Cougaar applications, user interfaces for the Pizza Application are available as web pages.

The main interface for the Pizza Party is the "/pizza" servlet, available at runtime at:
   http://localhost:8800/$Alice/pizza.

To see all the servlets associated with the party planner, Alice, navigate at runtime to the "/list" servlet:
   http://localhost:8800/$Alice/list
